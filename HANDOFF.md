# TimeTogether — Developer Handoff

> A no-account group availability scheduler. Friends drop in via a 6-character code or share-link, mark their busy hours with reason chips, and a heatmap shows when everyone is free.

This document captures everything needed to extend the app without re-deriving any design or product decision.

## 3. Architecture

### 3.1 Backend (`backend/server.py`)

A single FastAPI file. All routes are mounted under `/api`. Kubernetes ingress routes any path with `/api/` to the backend on port 8001.

| Method | Route | Purpose |
|--------|-------|--------|
| `GET`  | `/api/` | Health |
| `POST` | `/api/groups` | Create group → returns `{group, member_id}` |
| `GET`  | `/api/groups/{code}` | Read group (members, reasons, heat_colors) |
| `PUT`  | `/api/groups/{code}` | Rename group / update heat_colors |
| `POST` | `/api/groups/{code}/members` | Join group → returns `{member, member_id}` |
| `PUT`  | `/api/groups/{code}/members/{member_id}` | Rename a member |
| `PUT`  | `/api/groups/{code}/members/{member_id}/slots` | Replace member's slot list |
| `POST` | `/api/groups/{code}/reasons` | Add a busy reason |
| `DELETE` | `/api/groups/{code}/reasons/{reason_id}` | Remove a busy reason |

Single MongoDB collection: `groups`. Documents look like:

```jsonc
{
  "id": "<uuid>",
  "code": "ABC123",            // 6-char alphanumeric, no auth needed
  "name": "Weekend Warriors",
  "members": [
    {
      "id": "<uuid>",
      "name": "Alice",
      "color": "#1ABC9C",       // assigned round-robin from a 12-color palette
      "slots": [
        // default-free model: only BUSY slots are stored
        { "mode": "date", "key": "2026-05-08", "hour": 9,
          "status": "busy", "reason_id": "<uuid>|null" }
      ],
      "joined_at": "<iso utc>"
    }
  ],
  "reasons": [
    { "id": "<uuid>", "label": "Work", "color": "#7FB3D5" }
  ],
  "heat_colors": ["#F8FAFC","#D7F3EC","#9FE2D0","#38C29C","#F1C40F"],
  "created_at": "<iso utc>"
}
```

**Critical patterns:**
- Always project `{"_id": 0}` from MongoDB queries — `_id` is not JSON-serializable.
- Every document timestamp is stored as ISO string; we use `datetime.now(timezone.utc).isoformat()`.
- The 6-char code uses an unambiguous alphabet (no `0/O/1/I`).

### 3.2 Frontend

Two routes only: `/` (Landing) and `/g/:code` (Group dashboard). Identity per group is stored in `localStorage` under `tt:{CODE}:member_id`. Visited groups are tracked in `tt:groups` (deduped, capped at 20). Theme is stored in `tt:theme`.

**Default-free model** — a member is FREE for any (date, hour) unless an explicit `busy` slot exists in their `slots` array. The editor only persists busy slots; clearing a slot removes it from the array.

## 4. Component Map

### Pages

- **`Landing.jsx`** — hero, dual cards (Create / Join via code), how-it-works grid, decorative MiniHeatmap. Adds the new group to `tt:groups` on success and writes member-id to `tt:{CODE}:member_id`.
- **`Group.jsx`** — full dashboard. Holds all dashboard state:
  - `tab`: `"dates" | "members"` (default `"dates"`)
  - `editMode`: bool (toggles availability editor)
  - `rangeStart` / `rangeEnd` (ISO date strings)
  - `hourFrom` / `hourTo` (0–23)
  - `focusMemberIds: string[]` — multi-select compare mode
  - `now: Date` — refreshed every 60 s for live status bubbles
  - Computes `visibleMembers = focusedMembers || group.members` and pipes that into every child.

### Reusable components

| Component | Responsibility |
|-----------|----------------|
| `HeatmapGrid` | Read-only heatmap. Props: `members, columns, mode, hourFrom, hourTo, heatColors, focusMode, compareCount`. Single-member focus → cell colored by reason. Otherwise → `heatColor(freeCount, total, palette)`. Smart tooltip anchoring (`left-0`, `left-1/2`, `right-0`) based on column index. |
| `AvailabilityEditor` | Click+drag editor. Drag direction is set on mousedown to either "free→busy" or "busy→free". `editor-save-btn` PUTs the entire slot list. |
| `BusyReasons` | CRUD for reasons. Internal HSL hue/sat/light sliders + `hslToHex()` helper. |
| `QuickStats` | Computes Best overlap / Longest free streak / Top free hour-of-day from current view. |
| `LegendEditor` | 5-color picker for the heatmap palette. Saves to `PUT /api/groups/{code}` `heat_colors`. Reset restores defaults. |
| `GroupMenu` | Dropdown on the group title: rename, create new, switch to a previously visited group (from `tt:groups`). |
| `SuggestMeeting` | Picks top-3 distinct-day overlap windows, copies a friendly invite message. |
| `MembersSchedule` | Horizontal pill layout: `date · hour-range · reason` per busy run. Merges consecutive same-reason hours. |
| `ThemeToggle` | Light/Dark toggle. Adds/removes `html.dark`, persists in `tt:theme`. Respects `prefers-color-scheme` only on first visit. |

## 5. Styling System

### 5.1 Theme tokens (CSS variables in `index.css`)

Light defaults are on `:root`; dark overrides are on `html.dark`. Every component should reference these tokens — **do not hardcode `#fff` / `#0f172a`**.

```
--bg-base, --card, --card-soft
--ink, --ink-soft, --ink-mute, --btn-fg
--pastel-mint, --pastel-yellow, --pastel-lavender, --pastel-peach
--heat-0 (nobody), --heat-1, --heat-2, --heat-3, --heat-all (everyone)
```

A handful of Tailwind utility classes are remapped under `html.dark` (e.g. `.bg-white`, `.border-slate-900`, `.text-slate-700`) so legacy components don't need refactoring.

### 5.2 Design language — Neo-brutalist + soft pastel

- 2 px solid `var(--ink)` borders on cards & buttons (`.neo-card`, `.neo-btn`, `.neo-input`).
- Hard offset shadows (`box-shadow: 4px 4px 0 0 var(--ink)`), no blurry drops.
- Pill / rounded-xl shapes; pastel surfaces on solid white/dark backgrounds.
- Fonts: `Outfit` for headings (`.font-heading`), `Work Sans` for body. Loaded via Google Fonts in `index.css`.
- Text scale: `text-4xl sm:text-5xl lg:text-6xl` for h1, `text-base sm:text-lg` for h2, etc. (full table in `design_guidelines.json`).
- **Avoid:** purple-on-white gradients, generic Inter/Roboto, centered hero layouts, blurred shadows, emoji as icons.

### 5.3 data-testid convention

Every interactive or stateful element gets a `data-testid` in kebab-case. The full list lives in `test_reports/iteration_*.json` under `context_for_next_testing_agent`.

## 6. State Management & Data Flow

No Redux / Zustand / Context — local React state is enough.

```
Landing  ──POST /api/groups──▶  Backend
   │ writes tt:{CODE}:member_id, tt:groups
   ▼
Group.jsx (single source of truth)
   │ holds: group, focusMemberIds, tab, editMode, range, hour-range, now
   │
   ├─▶ HeatmapGrid       (reads visibleMembers + heat_colors)
   ├─▶ AvailabilityEditor (writes via updateSlots → setGroup mutation)
   ├─▶ MembersSchedule    (reads visibleMembers)
   ├─▶ QuickStats         (reads visibleMembers, computes derived stats)
   ├─▶ SuggestMeeting     (reads visibleMembers)
   ├─▶ LegendEditor       (writes heat_colors via updateGroup → setGroup)
   ├─▶ GroupMenu          (writes name via updateGroup or navigates)
   └─▶ MemberRow / LiveStatusBubble (toggles focusMemberIds, opens rename input)
```

After any mutation, the parent updates state in-place (`setGroup(g => …)`) — we do not refetch on every change. A full refetch only happens on mount and after `addVisitedGroup`.

## 7. Reusable Patterns

- **Default-free slot model** — never store `status: "free"` slots.
- **Slot map** — `buildSlotMap(slots)` returns a `Map<"mode|key|hour", slot>` for O(1) lookups inside grids. Always memoize.
- **Smart tooltip anchoring** — position class chosen from column index: `left-0` for first 2 cols, `right-0` for last 2 cols, otherwise center.
- **Multi-select focus** — `focusMemberIds: string[]`. Empty = full group; len 1 = single-member view (heatmap recolors busy by reason); len ≥ 2 = compare mode (heatmap uses normal palette but `total = focusMemberIds.length` so gold = "all selected free").
- **localStorage keys** — `tt:theme`, `tt:groups`, `tt:{CODE}:member_id`. Always namespace with `tt:`.
- **Hour-range filter** — `visibleHours = HOURS.filter(h => h >= hourFrom && h <= hourTo)`. Apply this in BOTH the heatmap and the editor; never pass the full `HOURS` array directly.
- **Date-mode columns** — `dateRange(start, end)` produces ISO strings; `formatDateShort()` produces `"Mon 8"`-style labels.

## 8. Build / Run / Deploy

See `/app/README.md` for one-command setup. Briefly:

```bash
# backend
cd /app/backend && pip install -r requirements.txt && uvicorn server:app --host 0.0.0.0 --port 8001

# frontend
cd /app/frontend && yarn install && yarn start
```

In the Emergent environment all of the above runs under supervisord — never start your own server. Hot-reload is enabled for both. After editing `.env` or installing deps, run `sudo supervisorctl restart backend` (or `frontend`).

Production deploy: use Emergent's Deploy button (managed) or push to GitHub and host frontend on Vercel + backend on Railway/Fly with a managed MongoDB.
